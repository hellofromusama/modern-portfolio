## Deviations from Plan

### Design / lint adjustments (no user permission needed)

**1. [Rule 3 - blocking] Internal CTAs use `next/link` `<Link>`**
- **Found during:** Task 4 (eslint gate)
- **Issue:** `<a href="/#projects">` (and `/contact`) tripped `@next/next/no-html-link-for-pages`; with strict lint enforced in `npm run build` this would fail the build.
- **Fix:** Converted the internal project CTAs to `<Link>` in `ProjectExperience.tsx` + the sr-only copy in `page.tsx` (matches the original project page which already used `<Link>`). Still real navigable anchors carrying `href` (smoke assertions unaffected).
- **Commit:** `245f12b`

**2. Nav labels shortened for HUD usability**
- Service-category HUD nav labels use short forms (Web / AI / Enterprise) instead of the long full category names; the full category names remain verbatim as the floated panel headings. Content text unchanged; only HUD chrome affected.

**3. Optional FAQ stop omitted**
- The plan marked the Services FAQ stop optional ("do not break tsc if omitted"). Omitted to stay surgical and avoid touching the guarded `space-dom.css` for section-bg transparency. Six evenly-spaced stops already fill the dive.

**4. Contact dive `scrollVh={420}`**
- The contact dive uses 2 stops, so a shorter scroll driver (420vh vs the 640 default) gives a tighter dive. `SpacePageShell` exposes `scrollVh` for exactly this.

Note: per the plan's `SpacePageShell` spec, each stop's planet shares the stop's `position` (planet glows behind the semi-transparent glass panel). This is the plan's explicit coupling, not a deviation.
